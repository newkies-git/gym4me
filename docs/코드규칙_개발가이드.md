## 3. Vue 컴포넌트 규칙

### 3.1 파일/컴포넌트 네이밍

| 유형 | 규칙 | 예시 |
|------|------|------|
| **View(페이지)** | `SomethingView.vue` | AuthView.vue, CalendarView.vue |
| **도메인 컴포넌트** | `DomainThingCard`, `DomainThingList`, `DomainThingModal` | GymCard, AddScheduleModal |
| **Base 컴포넌트** | `BaseXxx` | BaseCard, BaseModal, BaseTextField |

### 3.2 Composition API

- `<script setup lang="ts">` 사용
- `defineProps`, `defineEmits`에 타입 명시

```vue
<script setup lang="ts">
interface Props {
  isOpen: boolean
  title?: string
}
const props = withDefaults(defineProps<Props>(), { title: '' })

const emit = defineEmits<{
  close: []
  save: [value: string]
}>()
</script>
```

### 3.3 Props / Emits 네이밍

| 용도 | 규칙 | 예시 |
|------|------|------|
| Boolean 상태 | `isOpen`, `isLoading`, `isDisabled` | |
| 선택 상태 | `selectedGymId`, `activeTab` | |
| 컬렉션 | `items`, `options` | |
| v-model | `v-model:isOpen` → `update:isOpen` | |

### 3.4 Slots

역할 기반 슬롯 이름 사용:

- `header`, `toolbar`, `item`, `empty`, `loading`, `error`, `footer`

## 4. i18n 네이밍

### 4.1 기본 원칙

- **UI 텍스트 하드코딩 금지** (예외: 날짜/숫자 포맷, N 값)
- 키는 **의미(역할)**로 작성 (문구 변경에 강건)

```
Good: auth.resetPasswordLink
Bad:  auth.forgotYourPasswordQuestion
```

### 4.2 네임스페이스

- 도메인 단위: `auth.*`, `courses.*`, `gymTrainee.*`, `staffMgt.*`, `dashboard.*`
- 공통: `common.*` (close, save, cancel, loading)

### 4.3 하위 그룹

- `courses.detail.*`, `courses.actions.*`, `courses.toast.*`
- `auth.login.*`, `auth.signup.*`, `auth.reset.*`

### 4.4 동기화

- `ko.ts`와 `en.ts`의 **키는 1:1 유지**
- 신규 키 추가 시 **두 파일에 동시 추가**

## 5. CSS 네이밍 (BEM-lite + prefix)

### 5.1 원칙

- `scoped` 유지
- **컴포넌트 prefix** + BEM-lite

### 5.2 예시

```css
/* Block */
.courseDetail { }

/* Element */
.courseDetail__meta { }
.courseDetail__tag { }

/* Modifier */
.courseDetail__tag--warn { }
```

### 5.3 적용

- 대규모 일괄 변경 대신 **수정하는 파일부터 점진 적용**
- 미사용 CSS 제거

## 8. Pinia 스토어

### 8.1 네이밍

- 스토어 ID: 소문자 (`auth`, `scheduleStore`, `classStore`)
