# Gym4me 요구사항 대비 현재 구현 내용 비교 결과

사용자께서 제공해주신 새로운 메뉴 요구사항과 현재 구현된 시스템을 비교 분석한 결과입니다.

## 2. 상세 차이 분석 (Gaps)

### 2.1 캘린더 (Schedule)
- **보기 방식**: 현재는 이번 주(Sun-Sat) 7일만 고정으로 보여줍니다. 요구사항인 **2주 보기**와 **이전/다음 주 이동 버튼**이 추가되어야 합니다.
- **거절 사유**: 트레이너가 일정을 승인할 수는 있으나, 거절(Reject) 프로세스와 그에 따른 **거절 사유(Reason)** 입력 및 Trainee 알림 로직이 부족합니다.

### 2.2 기구 사용법 (Tool Usage) - **신규 모듈**
- 현재 시스템에 전혀 존재하지 않는 기능입니다. 
- **일반 정보(공개)**와 **개인 운동 내역(나/담당 트레이너만)**을 구분하는 복합 권한 필터링이 필요합니다.

### 2.3 매니저 알림 (Manager Alerts)
- 현재 대시보드에서 잔여 횟수가 부족한 회원을 보여주기는 하지만, 매니저가 특정 조건(예: 3회 이하 남음)으로 **회원 리스트를 추출**하거나 **직접 푸시/메일 알림**을 보내는 통합 관리 도구는 구현되어 있지 않습니다.

### 2.4 트레이너 프로필 (Trainer Profile)
- 현재는 간단한 자기소개(Bio)와 전문 분야(Specialties) 위주입니다. 요구하신 **경력 사항, 수상 이력** 등을 정형화된 필드로 관리하고 모두에게 공개하는 UI 개선이 필요합니다.

## 3. 향후 구현 제안

1.  **UI 개선**: 캘린더를 2주 단위 그리드로 확장하고 주간 네비게이션 추가.
2.  **기능 추가**: 'Tool Usage' 메뉴를 신설하고 Firestore에 관련 컬렉션 추가.
3.  **알림 시스템**: 매니저 전용 대시보드에서 알림 전송(Push/Email) 연동 (Firebase Cloud Messaging 또는 EmailJS 활용 가능).
4.  **모델 확장**: `users` 컬렉션에 트레이너 경력 사항 상세 필드 보완.
